CPSC 581 Project 1:
Rage Random Password

Overall Description

In a world where security threats on the internet are rapid and ever evolving one’s information needs to be protected. A good password can be the one thing between losing it all. And when you get hacked it can be an incredibly frustrating time that can make anyone mad.

This random password generator button allows the user to create passwords for use on any platform at the same time being an outlet of anger. The platform allows users to customize the password outcome while making the main interaction the button that generated the resulting password. Our team wanted to create simplistic utility for a real-life problem that many people face for project 1 in the Human-Computer Interactions II course at the University of Calgary.

Write-up for detailed sketches

This concept is the idea of the password generator in its simplest form and is a good example to build off from.

2. A password sketch showing the password result being shown in the middle of the page with customizations in the form of check boxes rather than another page. This keeps the interaction of regeneration and creation to a focused button. This concept is an iteration on to the first sketch.

3. A different interface for creating a password where choices are an up and down count to incorporate more password features while the creation button is at the bottom. This concept builds off the previous sketch.

4. This sketch generates the password with an animation on top and allows the user to copy the password by clicking the copy icon on the side in addition to regenerating the password with the same button on the bottom.

5. Interacting with the button at the bottom will create three random passwords at a time and these passwords can be copied by clicking the icon to the side or regenerated by clicking the same initial button. This concept is a spin on the last sketch to offer different options in a new way.

6. Interacting with the button will generate custom passwords that the user can adjust and regenerate to use on any platform. This concept is a combination of sketch 6 and 3.

7. This is fun idea where the user customizes their password features and then interacts with the button to generate many passwords that float around on the next page. The user must then double click on one of the passwords to copy for use. This concept is different from the other ideas while staying true to utility the team kept in mind.

8. This sketch showcases one option for customization for the password in the form a sliding scale to select the length of the password. Thus, the main interaction for the user is to create the password by clicking the button. This concept came to me when thinking about other options for password features and thinking about components of strong passwords.

9. This allows user fill customization of password features. This concept is a combination of sketch 8, 6, and 3.

10. The password in this sketch is generated with toggles instream of amount counters and creates three different passwords to select from. The main interaction is the button as this is how the user will achieve their goal of getting a password. This concept is a spin on sketch 3 and 5 using toggles and buttons only to keep the interaction focus on pressing the button for results. 

In the end, our group decided to collaborate on a sketch of to combine our ideas. Our combination of ideas resulted in sketch number 11. Sketch number 11 is a password generator that allows a user to increase the length of the password by repeatedly clicking the generate password button and customize the password by checking off the options to include upper letters, numbers, and special characters. So, when an a user is angry, button pressing will be a great outlet for emotions while solving the issue with a new password!
The idea is with more interaction with the button the stronger and more complex the password becomes, thus providing a great first line of defense against unauthorized access. The stronger the password, the more protected one’s computer will be from malicious entities. We ended up choosing this because the idea was a unique spin on the traditional password generator that provided the same real-life utility at the same time as being a creative use of button interaction. For the technical components, the project with constructed with best practice use of java script, css, and html. The team used all aspects in combination to create the working demo. I contributed to the base code to add the ability to add length of password characters through interaction with the button (about 25% of the total code) and sketched up the combined group idea of sketch 11 based off a combined brain storming session with group mates.
